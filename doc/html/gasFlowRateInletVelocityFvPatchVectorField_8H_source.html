28  Foam::gasFlowRateInletVelocityFvPatchVectorField
29 
30 Group
31  grpInletBoundaryConditions
32 
33 Description
34  Velocity inlet boundary condition either correcting the extrapolated
35  velocity or creating a uniform velocity field normal to the patch adjusted
36  to match the specified flow rate
37 
38  For a mass-based flux:
39  - the flow rate should be provided in kg/s
40  - if \c rho is "none" the flow rate is in m3/s
41  - otherwise \c rho should correspond to the name of the density field
42  - if the density field cannot be found in the database, the user must
43  specify the inlet density using the \c rhoInlet entry
44 
45 Usage
46  \table
47  Property | Description | Required | Default value
48  massFlowRate | mass flow rate [kg/s] | no |
49  rho | density field name | no | rho
50  rhoInlet | inlet density | no |
51  \endtable
52 
53  Example of the boundary condition specification for a mass flow rate:
54  \verbatim
55  <patchName>
56  {
57  type gasFlowRateInletVelocity;
58  massFlowRate 0.2;
59  rho rho;
60  rhoInlet 1.0;
61  value uniform (0 0 0);
62  }
63  \endverbatim
64 
65  The \c flowRate entry is a \c Function1 of time, see Foam::Function1Types.
66 
67 Note
68  - \c rhoInlet is required for the case of a mass flow rate, where the
69  density field is not available at start-up
70  - The value is positive into the domain (as an inlet)
71  - May not work correctly for transonic inlets
72  - Strange behaviour with potentialFoam since the U equation is not solved
73
